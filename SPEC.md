## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 個人工作室美髮/美睫/美甲師 | ~3 萬 | 客人散落在 LINE 對話、紙本筆記、IG 私訊，記不住每位客戶偏好 |
| 微型沙龍（2-5 位設計師） | ~1.2 萬 | 想做 VIP 經營但沒客戶資料系統；客戶流失率 30-50% |
| 中小型連鎖美業品牌 | ~800 | 多店管理、客戶跨店消費記錄追蹤困難 |

### 1.2 為什麼不做替代方案

- **紙本筆記本**：不易搜尋、無 Before/After 圖檔整合、客戶來一次就翻一次
- **Excel / Google Sheet**：能做但無照片附件、提醒功能、無 VIP 分析
- **商用 CRM（Salesforce/HubSpot）**：每月 1,500-3,000 NT$，對微型工作室太貴、太多用不到的功能
- **我們的解法**：純前端 SPA + IndexedDB，零月費、含 Before/After 照片、VIP 自動分析

## 2. 解決方案

### 2.1 核心價值主張

> 「客人上次做什麼、喜歡什麼、過敏什麼 — 不用翻對話紀錄，打開就知道。」

### 2.2 使用者流程

1. 新增客戶基本資料（姓名、電話、過敏史）
2. 每次服務結束後記錄消費項目 + Before/After 照片
3. 系統自動計算下次建議回訪時間（依服務週期推算）
4. 系統自動標記 VIP 客戶（累計消費 > 5 萬 或 月消費 > 4 次）
5. 到期前 LINE 通知設計師，主動提醒客戶預約

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 客戶檔案 CRUD（姓名、電話、生日、過敏史、備註）
- [ ] 服務歷程記錄（項目、日期、價格、設計師）
- [ ] Before/After 照片上傳 + 標註部位
- [ ] 自動回訪提醒（依週期推算下次日期）
- [ ] VIP 自動標記 + 消費排行榜

### 3.2 v2（加值）

- [ ] LINE 推播 API 整合（到期自動通知）
- [ ] 客戶消費趨勢圖表
- [ ] 多店管理（設計師分店切換）
- [ ] 客戶來源追蹤（IG/LINE/Walk-in）

### 3.3 明確不做

- 金流收款（沙龍現場收款為主，POS 整合複雜）
- 線上預約系統（與 Google Calendar 等外部服務衝突）
- 庫存管理（美業耗材複雜度高）
- 多語系介面（先做繁中、英文版 v3 再說）

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | React 18 + Vite + TypeScript | 與既有 41 個專案一致 |
| 狀態管理 | Zustand | 輕量、IndexedDB 整合容易 |
| 資料持久化 | IndexedDB | 支援大量照片 Blob、優於 localStorage |
| 圖片壓縮 | browser-image-compression | 上傳前壓到 < 500KB 節省空間 |
| 部署 | Vercel | 與其他專案統一 |

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| IndexedDB 大小限制（瀏覽器上限） | 🟠 中 | 照片自動壓縮、提醒使用者定期備份 JSON |
| 客戶資料遺失（裝置損壞） | 🔴 高 | 提供 JSON 匯出/匯入；v2 加雲端備份 |
| 個資法合規（客戶手機/照片） | 🟠 中 | 隱私頁明確聲明 + 加密儲存（v2） |
| 跨裝置同步需求 | 🟡 低 | v2 規劃雲端同步，先做單機版驗證需求 |
